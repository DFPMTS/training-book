本处默认大家已经了解了五级流水架构的CPU了。再次我规定大家了解的五级流水的架构是：
+ IF级:
    + 发起InstReq，然后IF阶段拿到了Inst
    + 更新PC寄存器的值是PC+4
+ ID级:
    + 对指令进行译码，得到各种控制码
    + 使用寄存器地址访问寄存器堆,并获取指令所需要的寄存器数据，会使用forward电路捕获Exe和Mem和Wb级的写回数据。
    + 计算出exe阶段所有需要的操作数
+ Exe级:
    + 使用操作数operand进行运算
    + 根据跳转指令的计算结果纠正PC寄存器的值
+ Mem级:使用Exe计算出的发起内存访问req，并等待内存的响应resp
    + 针对store裁剪和扩展好数据发送到内存
    + 针对load，对返回的数据进行裁剪和扩展
+ Wb级:根据写入寄存器的地址，将寄存器的数据写入到寄存器堆RegFile里面

在存在中断和异常情况下，我们要保证，指令不能修改CPU状态
CPU状态的定义是什么呢？我个人的定义是目前你的架构无法恢复的东西。这个是由你设计的架构决定的。一般来说寄存器堆和内存都会哪里CPU状态之中，当然你也可以设计架构去做到可以恢复，但是通常不轻松。

## 实现访存乱序？
再次重申:软件给定的程序其实是规定了两种访问顺序一种是寄存器另一种就是内存了，我们想要实现访问内存的乱序，就得和实现寄存器访问乱序一样去考虑这个问题。
首先回答:记分牌算法和Tomasulo算法是如何去解决这个问题？

我们在访存指令不能进行写违例检查，每次一个store执行完成都要拿他的地址去检查那些被推测指定load，是否存在地址相同，如果存在则要求那些load重新执行，即我们设计load和store推测执行。目前有个问题如果这些访存指令还是保存在rob里面会导致我们检查很麻烦，因为我检查的时候还要区分一个指令是不是访存指令，所以这里推荐将访存指令完成执行完成后保存到一个地方，集中管理。
